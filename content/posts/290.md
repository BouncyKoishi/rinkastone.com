<p>本文只整理与游戏逻辑有关的内容，忽略了UI，动画效果，画面渲染，音效等内容的相关实现。</p>



<p>人物/怪物身上的增益状态/异常状态在尖塔源码中被表述为<strong>Power(能力)</strong>。尖塔中每一个[能力]都由一个类规定其属性和行为，这些类继承一个共同的抽象类AbstractPower。</p>



<h3 class="wp-block-heading">基础概念/基础类结构</h3>



<p>此处只列举和[能力]有关的概念。</p>



<p><strong>Action(行为)</strong>: 即尖塔中存在的需要执行的特殊动作，例如抽牌[行为]、牌堆洗牌[行为]、伤害[行为]等。所有表示[行为]的具体类均继承抽象类AbstractGameAction。[行为]被创建后会进入行为队列中，由行为队列依次调用update()方法以执行这些[行为]。[能力]的创建、消灭属于一种[行为]，[能力]在发挥作用时可能创建各种类型的新[行为]。</p>



<p><strong>Dungeon(楼层)</strong>: 即尖塔中的楼层，存在塔底、城市、深处、终局四个具体类，它们继承抽象类AbstractDungeon。[楼层]类中的变量和方法基本可以视为本局游戏的全局变量，全局方法。行为队列是[楼层]实例中的一个元素。</p>



<p><strong>Creature(生物)</strong>: 即尖塔中存在的生物，存在[角色]和[怪物]两个抽象类，它们继承抽象类AbstractCreature。在游戏中，[能力]实例总是存在于[角色]或[怪物]实例下的能力列表中。</p>



<p><strong>Player(角色)</strong>: 即玩家角色，存在战士，猎手，机宝，观者四个具体类，它们继承抽象类AbstractPlayer，间接继承抽象类AbstractCreature。角色类中包含大量游戏基础信息，含血量、金币、卡组列表、遗物列表等。在游戏中，[角色]实例总是[楼层]实例中的元素。</p>



<p><strong>Monster(怪物)</strong>: 即敌方角色。所有表示[怪物]的具体类均继承抽象类AbstractPlayer，间接继承抽象类AbstractCreature。在游戏中，[怪物]实例总是存在于[房间]实例下的怪物列表中。</p>



<p><strong>Card(卡牌)</strong>: 即尖塔中存在的卡牌。所有表示[卡牌]的具体类均继承抽象类AbstractCard。在游戏中，[卡牌]实例通常存在于[角色]实例下的数个卡牌列表中。[卡牌]与[能力]存在多种交互关系，这些交互通常以[行为]作为中介。</p>



<p><strong>Relic(遗物)</strong>: 即尖塔中存在的遗物。所有表示[遗物]的具体类均继承抽象类AbstractRelic。在游戏中，[遗物]实例通常存在于[角色]实例下的遗物列表中。[遗物]可以创建新的[能力]，这种交互通常以[行为]作为中介。</p>



<p><strong>Potion(药水)</strong>: 即尖塔中存在的药水。所有表示[药水]的具体类均继承抽象类AbstractPotion。在游戏中，[药水]实例通常存在于[角色]实例下的药水列表中。[药水]可以创建新的[能力]，这种交互通常以[行为]作为中介。</p>



<h3 class="wp-block-heading">能力的基础属性</h3>



<p><strong>ID: </strong>[能力]的唯一标识，通常与[能力]的英文名称一致。</p>



<p><strong>名称:</strong> [能力]的展示名称，受国际化模块影响。</p>



<p><strong>所有者:</strong> 拥有该[能力]的[生物]。</p>



<h3 class="wp-block-heading"><strong>能力</strong>的创建或叠加</h3>



<p>[卡牌]的使用、[遗物]的生效、[能力]的生效、[怪物]的创建等事件可能会在某一时刻导致新[能力]的创建。此时发生的过程如下文所述。</p>



<p>ApplyPowerAction是一个表示“创建[能力]”这个[行为]的具体类。当需要创建[能力]时，尖塔创建一个新的ApplyPowerAction实例，并将该实例加入行为队列，由行为队列择机执行该[行为]。</p>



<p>ApplyPowerAction的实例创建需要提供以下的参数：</p>



<ul class="wp-block-list">
<li>行为创建者：创建这个新[能力]的[生物]。</li>



<li>行为目标：接收这个新[能力]的[生物]。</li>



<li>能力：被创建的[能力]实例。</li>



<li>能力层数：一个整数N，一般为正。当此能力是力量/敏捷/集中/幽魂形态时，N可以为负。</li>
</ul>



<p>ApplyPowerAction实例被执行时，检测行为目标当前已有的状态列表。若列表中存在与新[状态]相同的[状态]，向列表中的[状态]添加N层能力。否则，向列表中添加一个层数为N的新[状态]。</p>



<h3 class="wp-block-heading"><strong>能力</strong>的减少</h3>



<p>[能力]的生效（触发效果时，回合结束时）、[怪物]的行动（如乐嘉）等事件可能会导致[能力]层数的降低。此时发生的过程如下文所述。</p>



<p>ReducePowerAction是一个表示“降低[能力]层数”这个[行为]的具体类。当需要降低[能力]层数时，游戏创建一个新的ReducePowerAction实例，并将该实例加入行为队列，由行为队列择机执行该[行为]。</p>



<p>ReducePowerAction的实例创建需要与ApplyPowerAction相同的参数。特殊的，能力参数可以只提供[能力]的ID，而非能力实例本身。</p>



<p>ReducePowerAction被执行时，将目标能力层数降低K。如果K大于当前的能力层数，且此能力不为力量/敏捷/集中/幽魂形态时，向行为队列的顶端新增一个“消灭[能力]”的[行为]。</p>



<h3 class="wp-block-heading"><strong>能力</strong>的消灭</h3>



<p>[能力]的层数降为0、[卡牌]的使用（如异化）、[遗物]的生效（如圆石饰物）等事件可能会导致[能力]的消灭。此时发生的过程如下文所述。</p>



<p>RemoveSpecificPowerAction是一个表示"消灭[能力]"这个[行为]的具体类。当需要消灭[能力]时，游戏创建一个新的RemoveSpecificPowerAction实例，并将该实例加入行为队列，由行为队列择机执行该[行为]。</p>



<p>RemoveSpecificPowerAction的实例创建需要提供以下的参数：</p>



<ul class="wp-block-list">
<li>行为目标：拥有被消灭[能力]的[生物]。</li>



<li>能力：被消灭的[能力]实例。</li>
</ul>



<p>RemoveSpecificPowerAction被执行时，调用目标[能力]的onRemove()方法以执行该[能力]被消灭时所需的逻辑（如无实体被消灭时恢复受到的伤害），随后将该[能力]从其所有者的能力列表中移除。</p>



<p>此外，尖塔中还存在另一个表示"消灭[能力]"的[行为]的具体类RemovePowerAction。与RemoveSpecificPowerAction不同，RemovePowerAction通过[能力]的ID来查找并消灭目标[能力]，而非直接引用[能力]实例本身。当不确定目标[生物]是否拥有指定[能力]时，使用RemovePowerAction更为安全。</p>



<p>除上述两种[行为]外，以下情形也会导致[能力]的消灭：</p>



<ul class="wp-block-list">
<li>[生物]死亡时，其能力列表中的所有[能力]被消灭。</li>



<li>战斗结束时，[角色]能力列表中的所有[能力]被消灭。</li>



<li>回合结束时，部分[能力]因其自身的逻辑（如易伤在敌人回合结束时层数归零）而触发自身的消灭。</li>
</ul>



<h3 class="wp-block-heading"><strong>能力</strong>的效果执行</h3>



<p>AbstractPower中声明了一系列空方法，用于在游戏流程的特定时机被调用。每一个具体的[能力]类通过重写这些方法来实现其效果。这些方法在尖塔的Mod社区中通常被称为<strong>钩子(hook)</strong>。下文按照这些钩子被调用的游戏时机分类列举。</p>



<p><strong>回合相关钩子：</strong></p>



<ul class="wp-block-list">
<li>atStartOfTurn()：在其所有者的回合开始时被调用。例如充能在其所有者的回合开始时获得额外能量。</li>



<li>atEndOfTurn(boolean isPlayer)：在其所有者的回合结束时被调用。参数isPlayer表示其所有者是否为[角色]。例如猛毒在其所有者的回合结束时造成伤害并减少自身层数。</li>



<li>atEndOfRound()：在每轮结束时被调用。一轮指[角色]的回合和所有[怪物]的回合都结束。例如多层护甲在每轮结束时减少自身层数。</li>
</ul>



<p><strong>攻击相关钩子：</strong></p>



<ul class="wp-block-list">
<li>onAttack(DamageInfo info, int damageAmount, AbstractCreature target)：在其所有者发起攻击时被调用。例如荆棘在[生物]受到攻击时对攻击者造成反伤。</li>



<li>onAttacked(DamageInfo info, int damageAmount)：在其所有者被攻击时被调用。</li>



<li>wasHPLost(DamageInfo info, int damageAmount)：在其所有者因伤害而失去生命值后被调用。</li>
</ul>



<p><strong>伤害修改钩子：</strong></p>



<ul class="wp-block-list">
<li>atDamageGive(float damage, DamageInfo.DamageType type)：在计算其所有者造成的伤害时被调用，返回修改后的伤害值。例如力量通过此钩子增加其所有者造成的攻击伤害。</li>



<li>atDamageReceive(float damage, DamageInfo.DamageType type)：在计算其所有者受到的伤害时被调用，返回修改后的伤害值。例如易伤通过此钩子增加其所有者受到的攻击伤害。</li>



<li>atDamageFinalGive(float damage, DamageInfo.DamageType type)：在计算其所有者造成的伤害的最终阶段被调用，返回修改后的伤害值。此钩子在atDamageGive之后被调用。</li>



<li>atDamageFinalReceive(float damage, DamageInfo.DamageType type)：在计算其所有者受到的伤害的最终阶段被调用，返回修改后的伤害值。此钩子在atDamageReceive之后被调用。例如无实体通过此钩子将其所有者受到的所有伤害降为1。</li>
</ul>



<p><strong>卡牌相关钩子：</strong></p>



<ul class="wp-block-list">
<li>onPlayCard(AbstractCard card, AbstractMonster monster)：在其所有者打出[卡牌]时被调用。例如回声形态在其所有者打出[卡牌]时创建一张该[卡牌]的副本。</li>



<li>onAfterUseCard(AbstractCard card, AbstractCreature target)：在其所有者使用[卡牌]后（无论该[卡牌]是否被打出）被调用。</li>



<li>onCardDraw(AbstractCard card)：在其所有者抽到[卡牌]时被调用。例如静电放电在其所有者抽到攻击牌时向所有敌人充能。</li>
</ul>



<p><strong>能力相关钩子：</strong></p>



<ul class="wp-block-list">
<li>onApplyPower(AbstractPower power, int amount, AbstractCreature target)：在任何[能力]被施加时被调用。例如人工制品通过此钩子检测到负面[能力]被施加时，取消该[能力]的施加并减少自身层数。</li>



<li>stackPower(int stackAmount)：在相同ID的[能力]被叠加时被调用。默认实现为将层数增加stackAmount。部分[能力]重写此方法以实现特殊的叠加逻辑。</li>



<li>onRemove()：在此[能力]被消灭时被调用。部分[能力]需要在此钩子中执行善后逻辑。</li>



<li>onDeath()：在其所有者死亡时被调用。</li>
</ul>



<p>以上钩子的调用时机由游戏引擎在相应的游戏流程中统一触发。当某一时机到来时，引擎遍历当前场景中所有[生物]的能力列表，依次调用每个[能力]的对应钩子。这种设计使得每种[能力]只需关注自身关心的时机和逻辑，而不需要了解游戏流程的全貌。</p>
